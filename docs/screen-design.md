# 画面設計 — kaji-note

## 設計方針

- **画面数を最小限に**: 4 画面 + 1 モーダル + 1 Web 専用ページで全機能を網羅する
- **ノートの世界観**: 罫線・手書き風フォント・クリーム紙色を基調とし、実際のノートに書き込まれた手順書を連想させる UI とする
- **モバイルファースト**: 基準画面幅 375px、片手操作を前提とした要素配置

## S-02 ノート棚（ホーム）

**目的**: 手順書一覧の表示・検索・新規作成への導線

```
┌─────────────────────────┐
│  kaji-note          [*] │  <- header (group settings)
│  [Yamada Family  ▼    ] │  <- group switcher
│ ─────────────────────── │
│  [S] Search...          │  <- search bar
│ ─────────────────────── │
│                         │
│  ┌────────┐ ┌────────┐  │
│  │Bathroom│ │Laundry │  │  <- notebook spine cards
│  │[c] Tmrw│ │[v] Done│  │     (due date or done badge)
│  └────────┘ └────────┘  │
│                         │
│  ┌────────┐ ┌────────┐  │
│  │Cooking │ │Garbage │  │
│  │[!] Late│ │[c] Soon│  │
│  └────────┘ └────────┘  │
│                         │
│                    [+]  │  <- new button (fixed bottom-right)
└─────────────────────────┘
```

**要素:**
- ヘッダー: アプリ名 / グループ設定ボタン（⚙）
- グループ切り替えピッカー: 現在のグループ名を表示。タップでグループ一覧から切り替え可能（複数グループ所属時のみ表示）
- 検索バー（キーワード・カテゴリ絞り込み）
- ノートカード一覧（グリッド表示）
  - タイトル
  - 次回実施予定日 / 実施済バッジ / 期限超過バッジ
- 新規作成ボタン（╋、右下固定）

**カードバッジの種類:**

| バッジ | 意味 |
|--------|------|
| 🔴 期限超過 | 実施予定日を過ぎている |
| 🕐 〇〇日 | 次回実施予定日を表示 |
| ✅ 実施済 | 当日実施済み |
| （なし） | 頻度未設定 |

**グループ未所属時の空状態（初回サインアップ直後）:**

```
┌─────────────────────────┐
│  kaji-note          [*] │
│ ─────────────────────── │
│                         │
│   まだグループがありません   │
│                         │
│  ┌─────────────────┐    │
│  │ + グループを作成 │    │
│  └─────────────────┘    │
│                         │
│  招待リンクをお持ちですか？  │
│  リンクをタップして参加     │
│                         │
└─────────────────────────┘
```

- グループ切り替えピッカーは非表示
- `[+]`（新規作成）ボタンも非表示（グループなしでは手順書を作れない）
- 「グループを作成」ボタンは S-05 モーダルを開いて新規グループ作成を即時起動する

## S-03 手順ページ（閲覧）

**目的**: 手順書の詳細閲覧・実行モードへの切り替え

```
┌─────────────────────────┐
│  [<]          [↑][E]    │  <- header (back / share / edit)
│                         │
│  Bathroom Cleaning      │  <- title (handwritten font, large)
│  Weekly | Next: 5/3(Sa) │  <- frequency / next due date
│ ─────────────────────── │  <- ruled line
│                         │
│  1. Turn on the fan     │
│                         │
│  2. Spray detergent     │
│    ┌─────────────────┐  │
│    │      Photo      │  │  <- photo sticker style (border+shadow)
│    └─────────────────┘  │
│                         │
│  3. Scrub with brush    │
│                         │
│  ...                    │
│ ─────────────────────── │
│  ┌─────────────────────┐│
│  │   > Execution Mode  ││  <- run button (fixed footer)
│  └─────────────────────┘│
└─────────────────────────┘
```

**実行モード（画面内切り替え）:**

```
┌─────────────────────────┐
│  [X]          2 / 5     │  <- close / step progress
│ ─────────────────────── │
│                         │
│  2. Spray detergent     │  <- current step (large)
│                         │
│  ┌─────────────────────┐│
│  │        Photo        ││
│  └─────────────────────┘│
│                         │
│                         │
│  ┌────────┐ ┌─────────┐ │
│  │<- Prev │ │ Next -> │ │  <- step navigation
│  └────────┘ └─────────┘ │
│                         │
│  "Done" on last step    │  <- records completion date
└─────────────────────────┘
```

**要素（閲覧時）:**
- ヘッダー: 戻るボタン / 共有ボタン（↑）/ 編集ボタン（✏️）
  - 共有ボタンは作成者・admin のみ表示。タップでアクションシートを表示:
    1. 「グループに公開 / 下書きに戻す」（下書き ↔ 公開 の切り替え）
    2. 「閲覧専用リンクをコピー」（公開状態の場合のみ表示）
    3. 「手順書を削除」（赤色。実行前に確認ダイアログを表示）
- タイトル・頻度・次回予定日
- 公開状態バッジ（下書き状態の場合のみ「下書き」タグを表示）
- ステップ一覧（番号 + テキスト + 写真）縦スクロール
- 実行モード開始ボタン（固定フッター）

**要素（実行モード時）:**
- 閉じるボタン / ステップ進捗インジケーター
- 現在ステップのテキスト・写真（大きく表示）
- 前へ / 次へボタン
- 最終ステップで「完了・実施済みにする」ボタン
  - **公開ノート**: 活性化。タップで `executions` に記録し `next_scheduled_date` を更新
  - **下書きノート**: 非活性化（グレーアウト）。ツールチップ「公開してから実施を記録してください」を表示

## S-04 書き込み（作成・編集）

**目的**: 手順書の新規作成および既存手順書の編集

```
┌─────────────────────────┐
│  [X]              [Save]│  <- header (cancel / save)
│ ─────────────────────── │  <- ruled line
│                         │
│  Title: _______________ │  <- title input (on ruled line)
│                 [✨ 生成] │  <- AI generate all steps (UC-07-1)
│  Cat:   [Cleaning   v]  │  <- category picker
│  Freq:  [None       v]  │  <- frequency picker
│                         │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │  <- dashed line (step divider)
│  1. __________________  │
│     __________________  │  <- text input (on ruled line)
│                         │
│     [Cam] [Mic]  [AI]   │  <- photo / voice / AI buttons
│    ┌─────────────────┐  │
│    │  Photo preview  │  │  <- shown when photo attached
│    └─────────────────┘  │
│                         │
│ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
│  2. __________________  │
│     __________________  │
│     [Cam] [Mic]  [AI]   │
│                         │
│  + Add a step           │  <- add step button
│                         │
└─────────────────────────┘
```

**要素:**
- ヘッダー: キャンセル（✕）/ 保存ボタン
- タイトル入力欄
- タイトル横「✨ 生成」ボタン: タイトルを元にステップ一覧を AI 自動生成（UC-07-1）。既存ステップがある場合は確認ダイアログを表示
- カテゴリ選択ピッカー（掃除 / 料理 / 洗濯 / 収納 / その他）
- 頻度設定ピッカー（設定なし / 毎日 / 週次 / 月次 / 季節ごと / カスタム）
- ステップ（繰り返し）
  - テキスト入力欄（罫線上に直接書く感覚）
  - 写真追加ボタン（📷 カメラ / ライブラリ）
  - 音声入力ボタン（🎤）
  - AI 生成ボタン（✨）
  - 添付写真プレビュー（削除可）
- ステップ追加ボタン
- ステップ長押しで並べ替え（ドラッグ）・削除メニュー

**AI 機能の呼び出しタイミング:**

| ボタン | 動作 |
|--------|------|
| タイトル横「✨ 生成」ボタン | タイトルを元にステップ一覧を自動生成（UC-07-1） |
| ステップ内「✨」 | 既存ステップを文脈に次ステップを提案（UC-07-2） |
| ステップ内「📷」で写真添付後 | 写真から説明文を自動生成（UC-07-3） |
| ステップ内「🎤」 | 音声入力 → テキスト変換 → AI 整形（UC-07-4） |

```
┌─────────────────────────┐
│           [X]           │  <- close
│  ─────────────────────  │
│                         │
│  [@] Hanako Yamada      │  <- profile (tap to edit)
│                         │
│  ─────────────────────  │
│  Group: Yamada Family   │
│                         │
│  Members                │

│   - Taro Yamada         │
│   - Jiro Yamada         │
│                         │
│  ┌─────────────────┐    │
│  │ + Invite member │    │
│  └─────────────────┘    │
│                         │
│  ┌─────────────────┐    │
│  │ + New group     │    │  <- create a new group
│  └─────────────────┘    │
│                         │
│  ─────────────────────  │
│  Logout                 │
│                         │
└─────────────────────────┘
```

**要素:**
- プロフィール（アイコン・表示名、タップで編集）
- グループ名・メンバー一覧（役割表示）

## S-06 閲覧専用ページ（Web）

**目的**: グループ外ユーザーへの手順書の読み取り専用共有（ブラウザ閲覧）

**URL**: `{EXPO_PUBLIC_SHARE_BASE_URL}/share/{token}`  
**実装**: Expo Router の Web ビルド（`/share/[token]` ルート）が `get-note-by-share-token` Edge Function を呼び出し、ノートデータを表示する。

```
┌─────────────────────────┐
│  kaji-note              │  <- ヘッダー（アプリロゴのみ）
│ ─────────────────────── │
│                         │
│  Bathroom Cleaning      │  <- タイトル（手書き風フォント）
│  Weekly | Next: 5/3(Sa) │  <- 頻度・次回予定日
│ ─────────────────────── │
│                         │
│  1. Turn on the fan     │
│                         │
│  2. Spray detergent     │
│    ┌─────────────────┐  │
│    │      Photo      │  │
│    └─────────────────┘  │
│                         │
│  3. Scrub with brush    │
│                         │
│  ...                    │
│ ─────────────────────── │
│  アプリで開く            │  <- App Store / Google Play へのリンク
└─────────────────────────┘
```

**要素:**
- ヘッダー: アプリロゴのみ（ナビゲーション・ログインボタンなし）
- タイトル・頻度・次回予定日（読み取り専用）
- ステップ一覧（番号 + テキスト + 写真）縦スクロール
- 写真は `get-note-by-share-token` レスポンスの `storage_path` から署名付き URL を取得して表示（Edge Function が service_role で生成）
- フッター:「アプリで開く」リンク（App Store / Google Play への誘導）
- 編集・実施記録ボタンなし（閲覧専用）

**エラー状態:**
- トークン不正・無効化済み・下書き状態: 「このリンクは無効または期限切れです」メッセージを表示

## デザイントークン

| 項目 | 値 | 意図 |
|------|----|------|
| 背景色 | `#FAF6EE`（クリーム） | ノートの紙 |
| 罫線色 | `#C9C0A8` | ノートの罫線 |
| アクセントカラー | `#2A5FA5`（インクブルー） | ボールペンのインク |
| 危険アクション | `#C0392B`（赤） | 赤ペンの書き込み |
| タイトルフォント | 手書き風（Caveat / Klee One） | ノートのタイトル見出し |
| 本文フォント | ゴシック系（Noto Sans JP） | 可読性重視 |
| カード影 | `box-shadow: 2px 3px 8px rgba(0,0,0,0.12)` | 貼り付けた紙の浮き |
| 写真枠 | 白背景・1px ボーダー・2° 傾き | ノートに貼った写真 |
